## Relevant Context — Do Not Deviate Without Escalation

**Source: insight_engine.md §5.1**
The adapter publishes a catalog of **every** named value it produces: `name`, `tier`, `type`, `null semantics`, and `producer reference` (which adapter method produces it — for diagnostics, not engine routing). The engine validates at startup that every named value referenced by an active rule is present in the catalog with a compatible type. References to absent names fail loudly at startup, never silently at evaluation.

**Source: insight_engine.md §6.6 (via OTA-699, Code & Test Complete)**
Startup validation includes the input-catalog completeness check: a rule referencing a named value not in the catalog rejects the load. This catalog is the contract that check runs against.

**Source: migration plan (S3.11)**
Every named value the adapter produces is declared. This is the consolidation point — it must reflect the full producer set from OTA-714 (RAW), OTA-715 (DERIVED), OTA-716 (COMPUTED), OTA-717 (SMA/`sma_alignment`), and OTA-718–722 (gamma, ATR_14, IV percentile, chart_state, is_etf).

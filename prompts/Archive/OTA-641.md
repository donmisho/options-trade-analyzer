**SMA alignment narrative — deterministic rules.**

```
def sma_alignment(spot, sma8, sma21, sma50) -> str:
    bullish_stack = (sma8 > sma21 > sma50) and (spot > sma8)
    bearish_stack = (sma8 < sma21 < sma50) and (spot < sma8)
    max_spread_pct = (max(sma8, sma21, sma50) - min(sma8, sma21, sma50)) / spot * 100
    clustered = max_spread_pct < 0.5

    if bullish_stack:
        return "bullish stack — price above 8 > 21 > 50 SMA."
    if bearish_stack:
        return "bearish stack — price below 8 < 21 < 50 SMA."
    if clustered:
        return f"clustered — all three SMAs within {max_spread_pct:.1f}% of spot. Trend undefined."
    # mixed — describe where price sits relative to each SMA
    above = [name for name, val in [("8", sma8), ("21", sma21), ("50", sma50)] if spot > val]
    below = [name for name, val in [("8", sma8), ("21", sma21), ("50", sma50)] if spot <= val]
    return f"mixed — price below {' and '.join(below)}, above {' and '.join(above)}. Not a clean bullish or bearish stack."
```

The "mixed" case must render in the exact format shown by the QQQ v2 sample: `mixed — price below 8 and 50, above 21. Not a clean bullish or bearish stack.`

Add the SMA alignment rules to `business-rules.md` under a new "Technicals Classification" subsection.

**ETF short-circuit — non-negotiable.** Read `symbol_reference.is_etf` (or the equivalent column). If true:
- `Next earnings: N/A (<TICKER> is an ETF)`
- `Days to earnings: N/A`
- `Earnings in expiration window: No`
- **Never call the earnings provider for an ETF.** This is a defensive guard — an ETF would either return empty results or a misleading constituent's earnings date.

**Finnhub provider gating.** If `EarningsCalendarProvider` is not active (OTA-508 in flight) for non-ETF symbols:
- `Next earnings: unavailable (provider in flight under OTA-508)`
- `Days to earnings: unavailable`
- `Earnings in expiration window: unknown`

Do not emit bare `N/A` for missing provider — `N/A` is reserved for "not applicable to this symbol type." Missing provider is `unavailable` so the QA skill audit trail distinguishes the two states.

When OTA-508 ships and the provider becomes active, this code path automatically starts populating real values without changes — the gating check just flips.

**`Earnings in expiration window` rule.**
- ETF: always `No` (no earnings).
- Provider inactive: `unknown`.
- Provider active + earnings date present: `Yes` if `earnings_date <= expiration_date`, else `No`.

**No Claude API call.** All Technicals values are computed; all Earnings values are read or short-circuited. No Claude call in this code path (cost guardrail).

**Section ordering.** Technicals sits between Market context (OTA-640) and Earnings. Earnings sits between Technicals and Options chain (OTA-642). Source: QQQ v2 sample ordering.

**Same content for trades and positions.** Both export types render identically.

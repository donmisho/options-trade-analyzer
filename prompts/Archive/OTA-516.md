## Scope
In `app/analysis/strategy_scorer.py`, fix `score_all_strategies` (and any sibling scoring entry points that accept `user_config`) so that the nested per-strategy dict is unpacked correctly: each strategy's per-pass scorer receives only its own slice of overrides.

Reference implementation:

```python
def score_all_strategies(symbol_data, user_config=None):
    results = {}
    for strategy_key in STRATEGIES:
        strategy_specific_config = (
            user_config.get(strategy_key, {}) if user_config else {}
        )
        results[strategy_key] = score_strategy(
            strategy_key,
            symbol_data,
            user_config=strategy_specific_config,
        )
    return results
```

The per-strategy scorer continues to call `cfg.get("dte_min")` as it does today. The change is upstream at the dispatch point — the nested dict gets flattened to the strategy-specific slice before being passed in.

**Backward compatibility decision (defer to Phase 1 finding):**
- If grep finds zero callers passing the legacy flat shape, deprecate the flat shape outright. Cleaner.
- If any caller passes the flat shape, add a one-line shape-detection guard that treats a top-level dict with no strategy-id keys as the legacy flat shape and applies it uniformly (matching today's behavior). Document this in a code comment as a deprecation path.

Phase 1 reports the grep result before deciding.

## Relevant Context â€” Do Not Deviate Without Escalation

**Source: mcp-server-spec.md â†’ ADR-4 (Pass-Through to Existing Provider Adapters)**
Rule: Calls the existing `FinnhubEarnings` adapter (OTA-508). The adapter is the same one that powers the `EarningsInWindowGate` hard gate. No new earnings logic is added â€” only ETF detection and shape translation.

**Source: mcp-server-spec.md â†’ ADR-2 (Entra Resource Server)**
Rule: The `@mcp.tool()`-decorated function added by this Story is automatically gated by the `TokenVerifier` registered in OTA-605. The function receives a request context that includes the resolved `User` from the JWT `oid` claim â€” it does not re-validate auth and does not resolve users itself.

**Source: mcp-server-spec.md â†’ Tool Specifications â†’ get_earnings_date (equity)**
Tool shape: `{ ticker, next_earnings: { date, time, confirmed, source: "finnhub" }, days_until }`. `time` is `before_open`, `after_close`, or similar from Finnhub. `days_until` is calculated as the integer days between today and the earnings date.

**Source: mcp-server-spec.md â†’ Tool Specifications â†’ get_earnings_date (ETF)**
Tool shape: `{ ticker, next_earnings: null, is_etf: true }`. ETFs auto-satisfy the no-earnings-holds rule. The `is_etf` flag is informational; `next_earnings: null` is the contract.

**Source: business-rules.md â†’ No-earnings-holds rule (EarningsInWindowGate)**
Rule: Trades with confirmed earnings inside the holding window are filtered out by the hard gate. ETFs are automatically exempt because they don't report earnings. The `get_earnings_date` MCP tool exposes this same data so claude.ai conversations can apply the framework rule directly.

**Source: mcp-server-spec.md â†’ ETF detection**
Rule: ETF detection uses the existing `symbol_reference` table (8,568 rows, includes security type). **Do not** use a hardcoded list of ETF tickers. Pull the security type from `symbol_reference` and treat any non-equity, non-ADR security type as an ETF for the purpose of this tool.

**Source: mcp-server-spec.md â†’ Error Shape**
Rule: `EARNINGS_DATA_UNAVAILABLE` returned cleanly for tickers Finnhub does not cover (older or thinly-covered names).

**Source: mcp-server-spec.md â†’ Observability**
Rule: `tool_name=mcp.get_earnings_date`. Wraps the call in the `mcp_tool_observability` wrapper from OTA-605 with the resolved per-request `user_id`.

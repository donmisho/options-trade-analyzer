## OTA-509 — Scope (Subtask of OTA-549)
EV grounding rule. Block prose contradicting `computed_fields.expected_value` sign.

```python
def validate_ev_grounding(narrative_text, computed_fields):
    errors = []
    if computed_fields.expected_value < 0:
        if re.search(r"positive ev|favorable ev|ev of \$?\d", narrative_text, re.I):
            errors.append("EV_CONTRADICTION: narrative asserts positive EV but computed EV < 0")
    return errors
```

## OTA-510 — Scope (Subtask of OTA-549)
SMA grounding rule. Two sub-checks: positional contradiction + numerical hallucination.

```python
def validate_sma_grounding(narrative_text, computed_fields):
    errors = []
    # Sub-rule A: positional contradiction
    if computed_fields.price > computed_fields.sma_50:
        if re.search(r"below.*50|below.*all.*sma|under.*50", narrative_text, re.I):
            errors.append("SMA_POSITION: narrative asserts below SMA-50 but price > SMA-50")
    # Sub-rule B: numerical hallucination — SMA value cited but not in input set
    sma_values_in_input = {computed_fields.sma_8, computed_fields.sma_21, computed_fields.sma_50}
    sma_values_in_narrative = re.findall(r"sma[- ]?\d+\s+at?\s+\$?(\d+\.\d+)", narrative_text, re.I)
    for v in sma_values_in_narrative:
        nearest = min(sma_values_in_input, key=lambda x: abs(x - float(v)))
        if abs(float(v) - nearest) > 0.10:
            errors.append(f"SMA_HALLUCINATION: narrative cites SMA value {v} not in inputs")
    return errors
```

## OTA-510 — Acceptance criteria
- Both sub-rules run on every evaluation pre-emission.
- AMZN regression A: "below all three SMAs" against price > all three SMAs is blocked.
- AMZN regression B: cited "SMA-21 at 257.64" not in input set (with 10¢ tolerance) is blocked.
- Numerical tolerance: 10¢ window for floating-point match.
- Validation failures persisted to evaluation audit log.
- No false positives on narratives that correctly summarize positional context. Verify with at least two clean regression cases.

## OTA-510 — Out of scope
- SMA crossover signals (different domain).
- Other technical indicators beyond SMA-8 / SMA-21 / SMA-50.
- Retrofitting these checks onto historical evaluations stored in the database.

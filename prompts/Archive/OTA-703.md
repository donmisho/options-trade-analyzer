## Relevant Context — Do Not Deviate Without Escalation

Source: `insight_engine.md` §4.2 (result record + mandatory trace)
Rule: A result record is produced for **every** candidate (halted or completed). It carries the **mandatory full per-rule trace** — every decision, including non-stopping and zero-penalty failures — so the engine is auditable against the tables. Contents: candidate id + `candidate_type`; `source_app_id` + `strategy_key`; `terminal_phase`; per-gate results (rule_key, phase, tier, `evaluation_order`, value(s), params, pass/fail, `stop_if_fail`, `was_terminal`, held penalty, decision reason); per-criterion scoring breakdown (rule_key, raw value, weight, weighted contribution); raw score; held penalties applied; per-adjustment results (rule_key, amount, condition triggered, score before/after, reason); final adjusted score; verdict; engine metadata (config-version hash, engine version, run timestamp).

Source: OTA-695 OD-2 / OTA-703 ticket — **verdict_source discriminator**
Rule: Verdict is populated from exactly one of three paths, recorded on the record so consumers can tell them apart:
1. `BAND_LOOKUP` — completed scoring + adjustments; verdict from the strategy's `verdict_band_set`.
2. `HALT_TERMINAL_VERDICT` — a `stop_if_fail = true` gate failed and its junction `terminal_verdict` was non-null; verdict from the column, Phase-7 bypassed, `final_score` null (e.g. `WAIT_FOR_EARNINGS`).
3. `HALT_NO_VERDICT` — a stopping gate failed with null `terminal_verdict`; verdict null, `final_score` null.
`terminal_phase` is set in all three cases. The `verdict_source` field already exists on `ResultRecord` (OTA-697) — this Story **populates** it.

4. Set `terminal_phase` in all three cases. Stamp engine metadata (`source_app_id`, engine version, config-version hash, run timestamp).

## Verification steps
1. Completed candidate: assert `verdict_source = BAND_LOOKUP`, `final_score` non-null, full trace present.
2. Stopping-gate halt with `terminal_verdict`: assert `verdict_source = HALT_TERMINAL_VERDICT`, verdict from column, `final_score` null, `terminal_phase` = gate phase.
3. Stopping-gate halt without `terminal_verdict`: assert `verdict_source = HALT_NO_VERDICT`, verdict null, `final_score` null.
4. Non-stopping failure: assert the failed-gate decision is in the trace AND the candidate still reaches a verdict.
5. Assert `was_terminal` is true on exactly the halting decision (and false elsewhere).
6. Run the suite; OTA-696 scan passes.

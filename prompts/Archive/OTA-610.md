## Relevant Context — Do Not Deviate Without Escalation

**Source: UI-GUIDANCE.md § Part 4 (Display Formatting)**
All dates display as `mm-dd-yyyy` via the `formatDate()` helper. Never use locale strings (`toLocaleDateString`, etc.).

**Source: UI-GUIDANCE.md § Part 5 (Typography)**
All chart text and UI elements use monospace. Axis labels use 10px, `var(--muted)`, no bold.

**Source: UI-GUIDANCE.md § Part 7 (Buttons)**
Buttons and form controls are sized to content. Never `width: 100%` or stretched to fill. The new `Range` dropdown must match the existing `Periods` control's height, border, and padding exactly.

**Source: UI-GUIDANCE.md § Part 6 (SMA Chart)**
SMA chart has configurable moving averages and date range. Both controls are independent and user-selectable. Defaults: 8/21/50 day SMAs, 90-day range.

**Source: OTA-610 acceptance criteria — Range / Schwab interval mapping**
Use exactly these mappings. Do not introduce a custom aggregation layer.

| Range | Schwab interval | Approx bars |
|---|---|---|
| 7d | 30-min | ~65 |
| 14d | 30-min | ~130 |
| 30d | daily | ~21 |
| 60d | daily | ~42 |
| 90d | daily | ~63 (default) |
| 180d | daily | ~126 |
| 365d | weekly | ~52 |

**Source: OTA-610 acceptance criteria — SMA truncation**
SMAs recompute against the visible window. If the window does not contain enough lookback to compute a given SMA (e.g., a 7-day window cannot support a 50-day SMA), that SMA renders only for the bars where it has full lookback. Never render a partial or computed-from-truncated-data SMA line.

### Phase 1 — Backend (only if needed)

### Phase 2 — Frontend

**Range dropdown**

- New control in the chart header, immediately to the left of the existing `Periods` button
- Label: `Range`
- Options: `7d`, `14d`, `30d`, `60d`, `90d`, `180d`, `365d`
- Default: `90d`
- Visual style: match the existing `Periods` button exactly — same height, border, font, padding. Use existing CSS variables (`var(--bg2)`, `var(--border)`, `var(--text)`, `var(--muted)`). No inline hex.
- Selection lives in component state (`useState`). No localStorage in this Story.
- On change: refetch chart data with the new range and the corresponding Schwab interval.

**X-axis date labels**

- Render below the candle area, aligned with bars
- Format every label via `formatDate()` → `mm-dd-yyyy`
- Typography: 10px monospace, `var(--muted)`
- Density per range:
  - `7d` / `14d` (30-min bars): one label per trading day, positioned at that day's session-open bar
  - `30d` (~21 daily bars): every other bar
  - `60d` (~42 daily bars): every 5th bar
  - `90d` (~63 daily bars): every 7th bar (weekly cadence)
  - `180d` (~126 daily bars): every 14th bar (bi-weekly cadence)
  - `365d` (~52 weekly bars): every 4th bar (monthly cadence)
- Labels must not visually overlap. If two adjacent labels would overlap at the current chart width, reduce density further.
- Y-axis price labels remain unchanged.

**SMA truncation**

- Compute each SMA only from bars where full lookback is available
- The first `(period - 1)` bars of each SMA series are gaps — render no line for those positions
- For a 7d window with 30-min bars, only the 8-period SMA will have any rendered line; 21- and 50-period SMAs will be all-gaps. This is correct.

**Range capping**

- If the backend returns fewer days than requested (limited symbol history), render with the available data and display a small muted note in the chart header: `Showing N days (max available)` — 9px monospace, `var(--muted)`.

**Loading state**

- Refetch on range change uses the chart's existing loading pattern (whatever a `Periods` change already triggers). Match it exactly — do not introduce a new loading visual.

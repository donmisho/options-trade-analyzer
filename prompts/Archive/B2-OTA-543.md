## Relevant Context — Do Not Deviate Without Escalation

**Source: architecture-plan.md § Resource Shutdown Discipline (canonical)**
Every async resource opened by lifespan startup must be closed by lifespan teardown. Failure to close causes connection leaks, abandoned DB sessions, or partial mid-shutdown writes. The doc carries the explicit list of resources; this Story confirms or updates that list.

**Source: ticket OTA-543 → Three known issues**
1. `FoundryEvalAdapter` httpx client — created in `__init__`, held as `httpx.AsyncClient`, never closed during teardown.
2. `scheduler.shutdown(wait=False)` — allows process exit before in-progress jobs complete; mid-write Position Monitor runs are abandoned.
3. Token refresh background task — needs `task.cancel()` plus `await task` for graceful exit.

**Source: CLAUDE.md → Async credential rule**
`azure.identity.aio` async variants in async handlers. The token refresh task likely uses Azure credentials — confirm async usage during cancel.

**Source: CLAUDE.md → Fire-and-forget observability discipline**
Observability writes are fire-and-forget; observability failures must NEVER block primary operations. The shutdown sequence must not block on observability flushes.

## Phase 2 — Apply fixes

### Fix 1 — FoundryEvalAdapter shutdown

1. Add `async def aclose(self)` to `FoundryEvalAdapter` that calls `await self._client.aclose()` (use the actual attribute name from Phase 1).
2. In lifespan teardown, retrieve the adapter (likely via `app.state.ai_adapter` or the dependency-injection path) and `await adapter.aclose()`.
3. If multiple adapter instances exist, close all. Use the registry / factory if Story #1 (OTA-536) has unified the AI stack — note that this Story may run in parallel with C1; if C1 has not yet committed, just close the FoundryEvalAdapter from its current location.

### Fix 2 — APScheduler shutdown

1. Change `scheduler.shutdown(wait=False)` to `scheduler.shutdown(wait=True)`.
2. Verify the rest of teardown is OK with potentially blocking on running jobs (max job duration is < 30s typically — well under any infrastructure shutdown grace period).

### Fix 3 — Token refresh task graceful exit

1. Track the task on app state at startup: `app.state.token_refresh_task = asyncio.create_task(token_refresh_loop())`.
2. In teardown, after scheduler shutdown:
   ```python
   task = getattr(app.state, "token_refresh_task", None)
   if task and not task.done():
       task.cancel()
       try:
           await asyncio.wait_for(task, timeout=5.0)
       except (asyncio.CancelledError, asyncio.TimeoutError):
           pass
   ```

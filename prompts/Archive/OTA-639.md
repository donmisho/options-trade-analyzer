## Relevant Context — Do Not Deviate Without Escalation

**Legs table — v2 column set (in order).** Side · Type · Strike · Expiration · Qty · Bid · Ask · Mid · Delta · IV · Volume · OI.

Source: QQQ v2 sample Legs table header:
```
| Side | Type | Strike | Expiration | Qty | Bid | Ask | Mid | Delta | IV | Volume | OI |
```

`Mid` is computed at render time as `(bid + ask) / 2`, two decimals. `IV` is formatted `##.#%` (one decimal — matches the sample `17.4%`, `16.1%`). Volume and OI render as integers with thousands separators (`6,113`, `9,408`).

Leg data is read from the `option_chain_snapshots.chain_data` row matched to the trade's `(symbol, expiration, strike, type)`. If a leg row is not present in the snapshot → 4xx fail-fast with a diagnostic message (consistent with OTA-621's chain-snapshot fail-fast convention).

**Net metrics block — exact field set, in order.**

Credit spreads (`BULL_PUT_CREDIT`, `BEAR_CALL_CREDIT`):
- `**Entry credit:** $C.CC per contract ($T.TT for N contracts)` — emit signed convention as positive (credit received)
- `**Spread width:** $W.WW`
- `**Credit % of width:** X.X%`
- `**Max profit:** $C.CC per contract ($T.TT) — if <underlying> <comparator> $<short_strike> at expiration`
- `**Max loss:** $L.LL per contract ($T.TT) — if <underlying> <comparator> $<long_strike> at expiration`
- `**Breakeven:** $B.BB`
- `**R:R:** R.RR : 1`
- `**Underlying spot:** $S.SS`
- `**Cushion to short strike:** ±$X.XX (±Y.YY%)` — positive = cushion exists

Debit spreads (`BEAR_PUT_DEBIT`, `BULL_CALL_DEBIT`):
- `**Entry debit:** $D.DD per contract ($T.TT for N contracts)`
- `**Spread width:** $W.WW`
- `**Debit % of width:** X.X%`
- `**Max profit:** $M.MM per contract ($T.TT) — if <underlying> <comparator> $<strike> at expiration`
- `**Max loss:** $L.LL per contract ($T.TT) — if <underlying> <comparator> $<strike> at expiration`
- `**Breakeven:** $B.BB`
- `**R:R:** R.RR : 1`
- `**Underlying spot:** $S.SS`
- `**Cushion to breakeven:** ±$X.XX (±Y.YY%) — <semantic narrative>`

**Critical: cushion semantic split.**

- Credit spreads: `cushion = (underlying_spot − short_strike)` signed. Positive when there's cushion to the short strike. Label: `Cushion to short strike`.
- Debit spreads: `cushion = (underlying_spot − breakeven)` signed. **Negative values are legitimate** — for a bear put debit when price is above breakeven (unfavorable at entry), the cushion renders as `−$X.XX (−Y.YY%)` with a narrative tail like `— price is ABOVE breakeven at entry (unfavorable for bear put)`. Label: `Cushion to breakeven`.

Source: QQQ v2 sample line:
```
**Cushion to breakeven:** −$3.14 (−0.44%) — price is ABOVE breakeven at entry (unfavorable for bear put)
```

This semantic split is what the QA skill keys on. Do not unify the two cushion formulas under one label. Do not flip signs to make debit cushion always positive. Encode the formula and the narrative tail per spread type.

**`$` prefix exception — Net metrics only.** The QQQ v2 sample uses `$` prefixes inside the Net metrics block (e.g. `$3.95 per contract ($395.00 for 1 contract)`, `$10.00`, `$711.05`). This is the v2 sample's convention for the Net metrics block specifically; it overrides the general no-`$` house rule **inside this block only**. Every other block (header, Legs table cells, Greeks, Market context, Technicals, Earnings, score breakdown) follows the no-`$` rule.

Document the override in a code comment at the Net metrics builder so the next maintainer doesn't "fix" it.

**House style — formatting inside this Story.**
- Currency two-decimal: `##.00`
- Percent of width: one decimal `##.#%`
- Cushion percent: two decimals `##.##%`
- R:R: `R.RR : 1` (always two decimals on the ratio)
- IV (legs and spread mid): one decimal `##.#%` (matches sample)

**Greeks aggregation formula.** For each Greek g ∈ {delta, theta, vega, gamma}:

```
net_g = Σ_legs (side_sign × qty × leg_g)
where side_sign = +1 for long, −1 for short
```

Document the formula in a comment in the Greeks builder.

**Spread mid IV.** Quantity-weighted mean of leg IVs:

```
spread_mid_iv = Σ_legs (qty × leg_iv) / Σ_legs (qty)
```

Document the choice in a comment.

**IV Rank.** Reuse the value already computed by the scoring pipeline (do not recompute). Source the value from the `TradeRecommendation` row or the position's most recent scoring run, whichever is canonical.

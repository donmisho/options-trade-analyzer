## Relevant Context — Do Not Deviate Without Escalation

Source: `insight_engine.md` §6.6 (startup validation)
Rule: On **any** failure the loader emits a loud, structured error report and the engine **refuses to evaluate** any strategy (no partial/degraded evaluation). Required checks: rule references a named value not in the input catalog; junction references a missing `rule_id`/`strategy_id`; junction omits a parameter in the rule's schema; gate junction omits `evaluation_order` or `stop_if_fail`; a strategy's active scoring weights do not sum to 1.0 (within documented tolerance); a verdict band set is not monotonic; a `formula:<name>` is not in the registered library; a junction parameter violates the rule's type/bound; a named value's null semantics are incompatible with the rule's needs.

Source: OTA-699 ticket + `insight_engine-schema-ddl.md` §2 — **evaluation_order uniqueness (OTA-683 regression guard)**
Rule: No two enabled rules in the same `(strategy, phase)` may share an `evaluation_order`. The schema deliberately does **not** enforce this as a DB constraint (phase lives on the rule, not the junction), so it MUST be a load-time check. This is the exact defect class that produced the OTA-683 collision.

Source: OTA-695 OD-2 — **terminal_verdict domain check**
Rule: When a junction row's `terminal_verdict` is non-null, the value must be a member of the verdict-domain `engine_lookups` set for that strategy's `consumer_surface`. Null is allowed (most rows).

Source: OTA-695 OD-1 — **dual formula-registry validation**
Rule: The formula-membership check runs against **two** independent sources at load: (1) the SHARED `engine_lookups` set `('SHARED','formula_registry')` — answers "is this name in the contract"; (2) the **live registered rule library** — answers "does an implementation exist". Both reject the load independently with **distinct** messages. They must agree at load completion — drift (a name in only one source) is a **third** distinct error. Pre-Wave-3, the live-registry check runs against a **stub** registry; OTA-707 seeds both sources in sync per test.

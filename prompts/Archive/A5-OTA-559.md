## Phase 1 — Vertical scanner accepts per-strategy filter set

1. Find the vertical scanner entry point. `grep -rn "vertical_scan\|scan_verticals\|VerticalScanner" app/` (likely `app/services/scanner.py` or similar).
2. Read it end-to-end. Note the current call signature, the candidate emission loop, and where DTE is computed per candidate.
3. Add an optional parameter `enabled_strategies: set[str] | None = None`. When `None`, default to all four strategies enabled (preserves current behavior for callers that don't opt in).
4. For each candidate:
   - Compute DTE once (it should already be computed; don't double-compute).
   - Compute the set of strategies whose band includes this DTE. Use the canonical dict from Phase 0.
   - If the intersection of `enabled_strategies` and the matched-strategy set is empty, **skip emission**.
   - Otherwise, tag the candidate with the matched-strategy set (add a `fitting_strategies: list[str]` field on the candidate object or dict).

## Phase 2 — Per-strategy targeting

When the caller specifies a target strategy (e.g., `enabled_strategies={"WG"}`), the scanner returns only candidates whose DTE falls in that strategy's band. The `fitting_strategies` tag on each emitted candidate will then always include the target.

Update the call sites that scope to a single strategy:
- Strategy page → scan for trades for the active strategy → pass `enabled_strategies={current_strategy}`.
- Multi-strategy run from the main scanner → pass the user's enabled-strategy set from `user_config`.

`grep -rn "vertical_scan\|VerticalScanner" web/ app/` to find every call site.

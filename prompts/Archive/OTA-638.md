## Relevant Context — Do Not Deviate Without Escalation

**Spread type ENUM — source of truth.** Render exactly one of these four values, uppercase, no friendly names anywhere in the v2 output:

```
BULL_PUT_CREDIT
BEAR_CALL_CREDIT
BEAR_PUT_DEBIT
BULL_CALL_DEBIT
```

Source: OTA-638 Story "Relevant Context" + tradeexecutionskill.md Section 1.1 + the QQQ v2 sample line `**Spread type:** BEAR_PUT_DEBIT`.

This Story creates the centralized formatter. Every existing call site that renders a friendly string ("Bear Put Debit", "Bull Put Credit", etc.) inside the Export MD code path must be replaced. UI rendering of trade-type badges in `web/src/` is **out of scope** — that continues to use the existing title-case transform per UI-GUIDANCE.md Rule 17.

**DTE computation.** `(expiration_date − export_date_utc).days`. Both dates UTC. Emit as integer. Never `N/A` for an active trade or position; if the row lacks an expiration date that is a 4xx fail-fast (consistent with OTA-621's chain-snapshot fail-fast convention).

Source: OTA-638 Story + sample line `**DTE:** 18`.

**Date formatting.** All dates in the v2 output use `mm-dd-yyyy`. Dates with time use `mm-dd-yyyy hh:mm UTC`. House style — `claude_context/CLAUDE.md` House Style Rules. Backend renders via the equivalent Python helper (do not call the frontend `formatDate()` from the backend).

Lines that include a time in v2:
- `Exported: <ISO8601 timestamp>` — keep ISO 8601 (matches the sample line `**Exported:** 2026-05-11T18:24:13.965119+00:00`)
- `Last monitored: <mm-dd-yyyy hh:mm UTC>` — position exports only

Lines that are date-only:
- `Followed at: <mm-dd-yyyy>` (already present in v1; verify it stays correctly formatted)
- `Expiration: <mm-dd-yyyy>` inside the Trade structure block

**Strategy profile field.** Pulled from the position row (`positions.strategy_profile` or the equivalent column). Trade exports use the strategy profile that was assigned at evaluation time, persisted on the `TradeRecommendation` row. If the column does not exist for a record (legacy data), emit `Strategy profile: unassigned` rather than `N/A` — preserves audit trail.

Source: OTA-638 Story + sample line `**Strategy profile:** Steady Paycheck`.

**Current P&L.** Positions only. Format: signed two-decimal, no `$` prefix, sign always shown including `+0.00`. Trades emit `Current P&L: N/A` (a trade is not yet a position).

Source: sample line `**Current P&L:** +0.00`. House style: `##.00`, no `$` (CLAUDE.md House Style Rules + UI-GUIDANCE.md Part 4 formatting table).

**Last monitored line.** Positions only. Absent on trade exports. Format `mm-dd-yyyy hh:mm UTC`. Source: sample line `**Last monitored:** 05-11-2026 18:22 UTC`.

**v2 footer — exact text.** Place at the end of every v2 export (trades and positions). Replaces the v1 footer entirely.

```
*Generated by Options Analyzer for QA handoff via the `options-analyzer-qa` skill on claude.ai. Schema v2.0. Field labels are pinned to the v2 parse contract.*
```

Italicized via the leading and trailing asterisks shown above. Source: sample line at the bottom of the QQQ example.

**Schema version line — exact text.** Emit in the header block, immediately after `Exported:`:

```
**Schema version:** 2.0
```

Source: sample line in the header.

**House style invariants (do not deviate).**
- No `$` prefix on any monetary value anywhere in v2 output. Currency context is implied.
- Monetary `##.00` via two-decimal formatting.
- Scores `##.00` (this Story does not render scores; relevant for downstream Stories).
- Probabilities `##.00%` (not relevant to envelope; relevant downstream).

## Scope

1. **Schema version line.** Add `**Schema version:** 2.0` to the header block of both the trade export and the position export, immediately after the `Exported:` line.

2. **Strategy profile line.** Add `**Strategy profile:** <name>` to the header block of both exports. Read from the position row (positions) or the persisted recommendation row (trades). For legacy rows with no value, emit `Strategy profile: unassigned`.

3. **DTE in Trade structure.** Compute `(expiration_date − today_utc).days` and emit `**DTE:** <n>` in the Trade structure bullet list. Never `N/A` for active records — fail-fast 4xx if expiration is missing.

4. **Centralized spread_type ENUM formatter.** Add a single helper (e.g., `format_spread_type_enum(spread_type) -> str`) that returns one of the four uppercase ENUM strings. Wire every Export MD call site through it. Remove every friendly-string rendering inside the Export MD code path. Other code paths (frontend badges, UI tables) are out of scope.

5. **Current P&L line.** Positions emit `**Current P&L:** <signed ##.00>`. Trades emit `**Current P&L:** N/A`. Sign always shown including `+0.00`. No `$` prefix.

6. **Last monitored line.** Positions only. Emit `**Last monitored:** <mm-dd-yyyy hh:mm UTC>` in the header block. Trades omit this line entirely (not "N/A" — the line is not rendered).

7. **v2 footer.** Replace the existing v1 footer with the exact text in Relevant Context. Italicized. Same footer for trades and positions.

8. **Snapshot test fixture.** Add a snapshot test that exercises a known fixed-input fixture (one trade row and one position row) and asserts the v2 envelope block (header lines + footer) matches a committed expected output. Use the QQQ sample's envelope shape as the model for the position fixture.

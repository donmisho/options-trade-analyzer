## Background

This Story adds the two snapshot tools to the MCP server. The auth layer was pivoted from static bearer to Microsoft Entra OAuth 2.1 in OTA-605; this Story builds on that foundation. The tools themselves are auth-model-independent — they call the Schwab adapter and serialize results. The OAuth dance happens in the wrapping `TokenVerifier` from OTA-605, not here.

## Relevant Context — Do Not Deviate Without Escalation

**Source: mcp-server-spec.md → ADR-4 (Pass-Through to Existing Provider Adapters)**
Rule: MCP tools call existing internal capabilities, never duplicate them. The pattern: `_get_provider("market_data")` returns the Active Schwab adapter (Pattern 1) → tool calls the adapter's existing method → tool serializes the result into the documented JSON shape. No additional caching is added at the MCP layer.

**Source: mcp-server-spec.md → ADR-2 (Entra Resource Server)**
Rule: All `@mcp.tool()`-decorated functions added by this Story are automatically gated by the `TokenVerifier` registered in OTA-605. The tool functions receive a request context that includes the resolved `User` instance from the JWT `oid` claim — they do not re-validate auth and they do not resolve users themselves.

**Source: architecture-plan.md → Pattern 1 (Provider Adapter Layer)**
Rule: All market-data calls go through `_get_provider("market_data")`. Hardcoding the Schwab adapter is a code-review fail.

**Source: mcp-server-spec.md → Tool Specifications → get_quote**
Tool shape: `{ ticker, price, bid, ask, volume, prev_close, change_pct, timestamp, market_state }` where `market_state` is one of `PRE | REGULAR | POST | CLOSED`. Source method: `SchwabMarketData.get_quote()`.

**Source: mcp-server-spec.md → Tool Specifications → get_option_chain**
Tool shape: `{ ticker, underlying_price, expiration, dte, options[] }` where each option has `{ occ_symbol, strike, type, bid, ask, mid, last, volume, open_interest, iv, delta, gamma, theta, vega }`. Default `strike_range_pct` is `0.15`. Default `option_type` is `"both"`. Source method: `SchwabMarketData.get_chain()`.

**Source: mcp-server-spec.md → Error Shape**
Rule: Errors return as structured response bodies, not stack traces. HTTP status remains 200 for tool-level errors so claude.ai can read the error and respond intelligently. Codes used in this Story:
- `TICKER_NOT_FOUND` — Symbol not in Schwab's universe
- `EXPIRATION_INVALID` — Date format wrong or expiration not available for ticker
- `SCHWAB_UNAVAILABLE` — Upstream Schwab API down or rate limited

**Source: mcp-server-spec.md → Observability**
Rule: Each tool call writes one row to `agent_run_log` via the `mcp_tool_observability` wrapper updated in OTA-605. `tool_name` values: `mcp.get_quote`, `mcp.get_option_chain`. The `user_id` is the resolved per-request user from the JWT `oid` claim. Failed observability writes never block the primary response.

**Source: SCHWAB-LOGIN-PROCESS.md**
Rule: Schwab access tokens are managed by `SchwabTokenManager`; the adapter is opaque to callers. The MCP tool does not touch the token directly.

### Phase 2 — Build (after STOP gate)

1. In `app/api/mcp_routes.py`, register `@mcp.tool() async def get_quote(ticker: str) -> dict` that:
   - Receives the resolved `User` from the MCP request context (per the OTA-605 pattern)
   - Resolves the market data provider via `_get_provider("market_data")`
   - Calls `provider.get_quote(ticker)`
   - Translates the response into the documented MCP shape (handle field-name/unit translations identified in Phase 1)
   - Returns the structured error shape on `TICKER_NOT_FOUND` and `SCHWAB_UNAVAILABLE`
   - Wraps the call in `mcp_tool_observability("mcp.get_quote", ticker=ticker, user_id=user.id, ...)`

2. In the same module, register `@mcp.tool() async def get_option_chain(ticker: str, expiration: str, option_type: str = "both", strike_range_pct: float = 0.15) -> dict` that:
   - Receives the resolved `User` from the MCP request context
   - Resolves the provider the same way
   - Calls `provider.get_chain(ticker, expiration)`
   - Filters strikes to `[underlying_price * (1 - strike_range_pct), underlying_price * (1 + strike_range_pct)]`
   - Filters by `option_type` (`call`, `put`, or `both`)
   - Translates each option into the documented MCP shape
   - Returns the structured error shape on `TICKER_NOT_FOUND`, `EXPIRATION_INVALID`, and `SCHWAB_UNAVAILABLE`
   - Wraps the call in `mcp_tool_observability("mcp.get_option_chain", ticker=ticker, user_id=user.id, ...)`

3. Write clear docstrings on both functions — these become the tool descriptions claude.ai sees and uses to decide when to call each tool.

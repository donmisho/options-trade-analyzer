### Update _get_provider() to honor state

Modify the dispatch helper:
- `state == "active"` → routable, normal selection
- `state == "inactive"` → routable only when explicitly requested by config (e.g., `settings.preferred_provider == "<key>"`)

- `state == "removed"` → not routable; raise immediately with a clear error pointing at the migration_target if present

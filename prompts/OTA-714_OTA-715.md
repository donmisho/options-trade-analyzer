## Relevant Context — Do Not Deviate Without Escalation

**Source: insight_engine.md §5 / §3.5 (tiers)**
RAW = produced directly from the data source (Schwab chain). DERIVED = computed from RAW without external calls; cheap, deterministic, produced by the adapter. The adapter does not run rules, score, or know strategies.

**Source: rules-engine-audit-2026-05-20.md §5d (DERIVED table) — verify against the audit doc in Phase 0**
DERIVED producers in scope for OTA-715: `DTE`, `net_debit`/`net_credit`, `spread_width`, `max_profit`, `max_loss`, `breakeven`, `prob_of_profit` (from delta), `ev_raw`, `reward_risk_ratio`, `cushion_pct`, `bid_ask_spread_pct`, `premium_dollars`, `theta_runway_days`, `credit_pct_of_width`, `debit_pct_of_width`. Each is computed from RAW chain values only — no Black-Scholes, no external calls (B-S is COMPUTED, OTA-716).

**Source: CLAUDE.md — Provider routing**
Schwab chain access routes through `_get_provider()` / settings via the `_shared/` Schwab client provider established in OTA-713. Never hardcode the provider.

**Source: insight_engine-migration-plan.md (S3.2)**
After the move, `analysis_routes.py` is left with a single call: `adapter.produce_candidates(scan_request)`. No chain-fetching logic remains in the route.

**Source: CLAUDE.md — async credential rule**
Any Azure access in async paths uses `azure.identity.aio`; no module-level `DefaultAzureCredential()`.

**Source: house style**
Probabilities format `##.00%`; no `$` prefix on monetary values; monetary `##.00`. (These are presentation rules — relevant only if any producer formats output; producers should emit raw numerics, not formatted strings.)
